## 1. 터빈 출구 온도(T₄) 판정기준 모순 (Issue #2 항목 2)

문제: 논문 초안 표에서 판정기준을 "T₄ < 923 K"로 명시했으나, 시뮬레이션 결과는 924.87 K로
기준을 초과함에도 합격(체크) 표시됨. 오차 0.2%는 근접도일 뿐 합격 근거가 될 수 없음.

원인: 이론 예측값 자체가 약 925 K이므로, 기준을 923 K로 설정한 것이 애초에 오기.

```
T₄ = T₃ + ΔT_comp × η_m × (1 + FAR)
   = 1250 + (−325.6) × 0.98 × 1.019
   ≈ 924.85 K
```

권장 수정안(택1). 옵션 A: 기준값을 이론 예측값에 맞춰 수정 — 기준 ~925 K, 결과 924.87 K,
판정 합격. 옵션 B: 기존 "< 923 K"를 유지하되 오차 허용범위를 명시 — 기준 925 K 이하(이론값 대비
±2 K 허용), 결과 924.87 K, 판정 합격. 두 옵션 모두 표에는 기준이 시뮬레이션 결과가 아니라
이론식에서 나왔다는 점을 각주로 명시할 것.

## 2. 터빈 동력 계산 — 질량유량 일관성 재검토 (Issue #2 항목 4)

문제 제기: 온도식 T₄ = T₃ + ΔT_comp·η_m·(1+FAR)는 연료 질량 증가(1+FAR)를 반영하는데,
동력식 W_turb = ṁ·cp·(T₃−T₄)는 ṁ = ṁ_air = 3.493 kg/s를 그대로 사용해 (1+FAR)을 반영하지 않음
→ 내부 불일치로 지적됨.

독립 재검산 결과(본 문서에서 직접 계산):

| 가정 | 질량유량 | W_turb 계산값 | VHDL 결과(1.38691 MW) 대비 오차 |
|------|----------|---------------|-----------------------------------|
| ṁ_air 그대로 (FAR 미반영) | 3.493 kg/s | 약 1.377 MW | 0.71% |
| ṁ_gas = ṁ_air×(1+FAR) (FAR 반영) | 3.560 kg/s | 약 1.404 MW | 1.24% |

(공통: cp_gas ≈ 1212.7 J/(kg·K), T₃−T₄ ≈ 325.13 K)

해석: Issue #2가 제안한 "Fix 6"(FAR 반영 질량유량 사용)을 그대로 적용하면 오차가 0.71%에서
1.24%로 오히려 커집니다. 즉 VHDL의 실제 turbine1.power 출력은 ṁ_air(FAR 미반영) 가정과 더 잘
맞습니다.

결론 및 권고: VHDL 모델은 온도 계산 단계에서는 연료 질량 증가를 반영하지만, 동력 계산 단계에서는
반영하지 않는 것으로 보이며, 이는 오류라기보다 모델의 단순화(simplification)로 판단됩니다.
논문에는 "Fix 6"처럼 값을 바꾸는 대신, 다음과 같이 한계점(limitation)으로 명시할 것을 권장합니다.

한계점 서술 예시: "본 VHDL-AMS 모델은 터빈 동력 계산 시 연료 질량 증가(1+FAR ≈ 1.019)를 반영하지
않고 공기 질량유량(ṁ_air)만을 사용한다. 이로 인한 영향은 약 1.9% 수준(약 26 kW)으로, 설계점 수준
검증에서는 무시 가능하나 정밀 사이클 해석 시 유의가 필요하다."

## 3. 노즐 면적 역산 로직 모순 (Issue #2 항목 3) — 논문 원고 전용, 저장소에 해당 코드/컴포넌트 없음

저장소의 VHDL 5개 컴포넌트(Inlet/Compressor/Combustor/Turbine/Shaft)에는 노즐(Nozzle)이
포함되어 있지 않아, 4.2/4.4절에서 언급된 노즐 면적 역산 로직은 이 저장소에서 직접 확인/재계산이
불가능합니다. Issue #2 코멘트의 권장 수정 문구를 참고용으로 남깁니다(팀원이 논문 원고에서 직접
검토 필요).

문제: "p_e = p_in(=대기압) → 유량함수 = 0 → 유량 0"이라고 인정한 직후, "mdot ∝ area(선형
비례관계 성립)"이라고 결론지어 자기모순처럼 보임. 4.4절의 area=0.0637 m² → mdot=18.92 kg/s
역산 결과와도 상충.

권장 수정 방향(논문 텍스트에 반영): "해면고도 정상 작동 시 터빈 출구압력은 대기압보다 높다.
실제 p_in(터빈 출구) > p_back(=p_amb=101,325 Pa)이므로 압력차가 존재하며, 이 조건에서 아음속
유량식을 적용하면 mdot ∝ area가 성립한다. p_e = p_in인 조건은 유량이 0이 되는 이상적 한계
조건(압력차 소멸 시점)이며, 실제 모델에서는 터빈 출구압이 더 높아 유량이 정상적으로 발생한다."

이 항목은 저장소에 노즐 모델 파일이 추가되기 전까지는 논문 텍스트 수정만으로 해결해야 합니다.

## 4. 시뮬레이션 구간(200 s) 타당성 (Issue #2 항목 5) — 후속 조사 필요

문제: 200초 구간 사용에 대한 정량적 근거(시상수 τ=J/b 계산, 정상상태 수렴 판정기준, 실제 운용
맥락)가 논문에 없음.

저장소에서 확인 가능한 사실: `components/05_Shaft/README_Shaft.md`에 명시된 대로 축은
관성모멘트 J=1.0 kg·m²의 1차 ODE(J·dω/dt = W_turb−W_comp−W_prop)로 모델링되어 있습니다.
그러나 축 부하(W_prop, 프로펠러 부하 토크 특성)에 대한 구체적 함수형이 저장소에 문서화되어 있지
않아, 본 문서에서 시상수 τ를 독립적으로 계산할 수 없습니다.

권장 조치(팀원 확인 필요). ① Twin Builder 결과 그래프(`components/0*/images/*.png`)에서 각
신호가 정상상태에 도달하는 실제 시각(예: 63.2%, 95%, 99% 도달 시각)을 판독하여 실효 시상수를
역산해 표로 정리. ② "정상상태 수렴" 판정기준을 정량적으로 명시(예: 마지막 10 s 구간 신호 변동률
< 0.01%). ③ UAV 순항 임무 프로파일 상 해당 조건이 200 s 이상 유지되는지 임무 프로파일 근거 자료
인용.

## 6. TIT(T₃) 구조적 고정값 — 이론 평형 재계산

연소기 에너지평형식:

```
T₃ = T₂ + η_b · FAR · LHV / [cp_air · (1 + FAR)]
```

| 파라미터 조합 | η_b | FAR | 계산된 T₃ | VHDL 고정값 대비 차이 |
|---------------|-----|-----|-----------|------------------------|
| VHDL 실제값 | 0.96 | 0.019 | 약 1,380 K | +130 K |
| 문헌 기준값 | 0.990 | 0.018 | 약 1,364 K | +114 K |

(cp_air 약 1004.6 J/(kg·K), LHV = 43,000,000 J/kg[ASTM D1655 Jet-A], T₂ = 613.75 K)

두 파라미터 조합 모두 VHDL 고정값(1,250 K)보다 100 K 이상 높게 산출되므로, TIT는 모델이 도출하는
출력이 아니라 부여된 설계 목표값입니다. 이는 오류가 아니라 VHDL 모델의 구조적 특성이며, 논문
방법론 섹션에 명시적으로 기술할 것을 권장합니다(`components/03_Combustor/README_Combustor.md`에
1차 안내문 반영 완료).

## 7. Ansys port in quantity 구조적 제약 (Issue #2 항목 11) — 공식 근거 미확인

문제: "VHDL-AMS port in quantity는 Twin Builder Results에서 0으로 표시되는 구조적 제약"이라는
주장에 대해 Ansys 공식 문서 인용이 없음. 역산으로 1,250 K를 증명하는 논리 자체는 타당하나, 제약의
근거가 없으면 심사 과정에서 문제 제기될 수 있음.

권장 조치: Ansys 공식 포럼/Help 문서에서 VHDL-AMS quantity의 in/out 포트 결과 표시 제약에 대한
근거 문서를 확인해 각주로 인용. 확인 전까지는 "관찰된 동작(observed behavior)"이라는 표현으로
서술 수위를 낮출 것을 권장.
